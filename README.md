# HCS
HCS(Holy Code Style) - методология разработки ПО(программного обеспечения), которая основывается на словах великого программиста UserPointer.

## Основные концепции
1. Идентификатор должен начинаться с заглавной буквы.

Пример:
```C
int Variable = 100;

struct C {
    int X;
    int Y;
}

int Func(int X, int Y) {
    return X + Y;
}
```
2. Старайтесь все всегда комментировать.
3. Функции могут быть любого размера.
4. Не используйте рекурсию.
5. Используйте ```goto``` если необходимо.
6. Не используйте нынешние концепции OOP.

Пример:
```c
// Используйте обычную сишную структуру вместо классов.
struct C {
    int ParameterOne;
    double ParameterTwo;
    // ...
}
```
7. Не используйте в идентификаторах цифры пишите их словами.

Пример:
```c
int VariableOne = 100;
int VariableTwo = 200;
```
8. Используйте часто макросы
9. Заменяйте условие ```x =! y && y != b``` на ```!(x == y && y == b)```
10. Не используйте лямбда функции
11. Не используйте анонимные функции
